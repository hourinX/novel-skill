# 小说大纲

> 本文件是 AI 写章节的核心依据。每一行就是 `generate --chapter N` 时 AI 看到的"剧本"。
> **格式必须是 7 列的 markdown 表格**，否则解析失败（详见 `ai_service/core/outline.py`）。
> 续写大纲可以用 CLI 的 `extend-outline` 命令交给 AI 自动补，也可以手写下面的表格。

---

## ⚠️ 最重要的一条规则

**正式使用时，本文件只需要保留下面这张 7 列表格**（表头 + 分隔行 + 至少 1 行章节数据）：

```markdown
| 章节  | 标题 | 核心事件 | 人物弧光 | 伏笔 | 情绪基调 | 结尾钩子类型 |
|-------|------|----------|----------|------|----------|------------|
| 第1章 | 雨夜的最后一根稻草 | 17 岁的 [主角姓名] 在 [所在城市] 的旧公寓中，因家中变故被迫接手父亲未完成的工作。当晚他独自处理一件本不该由少年承担的事情，过程中第一次见到了 [关键对手/秘密线索]，并在精疲力尽时被一位陌生人救下。对方留下一句意味深长的话便消失在雨中。 | 麻木认命 → 被现实碾压 → 第一次想"凭什么是我" | 救他的人手腕上有一枚刻着 [某种符号] 的旧戒指，与父亲遗物上的图案完全一致 | 阴冷沉闷 → 转为压抑的不甘 | 悬念型：他握紧手中那枚陌生人遗落的物件，雨声里第一次听见了自己的心跳 |
```

为什么这么严格：
- 解析器只识别形如 `| 第N章 | ... |` 且至少 7 列的行（见 `ai_service/core/outline.py` 的 `_CHAPTER_ROW_RE`）
- 但 `read_outline()` 会把**整个文件原文**塞进 AI 的 prompt 上下文。所以下方的「字段说明」「续写建议」等帮助文字也会被送进去，会污染 prompt
- 真正开始写书前，请把"字段说明 / 章节表 / 续写建议"等说明性章节**删除**，只留 7 列表格
- 本仓库为方便新人，默认把说明都写在了下面，请按需精简

## 📝 章节也能自动生成

不想自己一行一行写表格？用 CLI 的 `extend-outline` 让 AI 接着写：

```bash
extend-outline                                # 自动补齐到下一个 5 的倍数章节
extend-outline --from 10 --count 5            # 从第 10 章之后续写 5 章
extend-outline --note "主角进入魔界，遇到劲敌" # 指定续写方向
```

> AI 续写时会读取 worldview / character / memory / 已有章节正文，尽量与前文风格一致。
> 续写完之后**人工通读**，删掉不合适的行再 `generate`，避免崩节奏。

## 字段说明

| 列名 | 写法要点 |
|------|---------|
| **章节** | 必须形如 `第1章 / 第12章`，章节号必须连续，不能跳号 |
| **标题** | 6-12 字，要有意象或矛盾感，不要写"第一次出门"这种白描 |
| **核心事件** | 80-200 字，写"主角做了什么 + 遇到了谁 + 结果是什么"，AI 会基于这一列展开成几千字 |
| **人物弧光** | 主角心境的开始 → 转折 → 结束，用箭头表示，例如"麻木 → 受刺激 → 决意反击" |
| **伏笔** | 这一章要埋下的细节（一句话），后续章节会自动回收。可空，但建议每 2-3 章至少埋 1 个 |
| **情绪基调** | 给 AI 的写作色调，例如"压抑沉重 → 转为孤勇"。两段式比单一词好 |
| **结尾钩子类型** | 命名钩子类型 + 一句话描述，常见类型：悬念型 / 命运型 / 情感型 / 反转型 / 危机型 |

## 续写建议

- 前 5 章打基础：身份、动机、世界规则、第一个对手、第一次失败/觉醒
- 每 5 章一个小高潮，每 20 章一个大转折
- 写不下去时跑 `extend-outline --note "希望往哪个方向发展"`，让 AI 帮你续 5 章
- AI 续写偶尔会偏离风格，记得人工通读、删掉不合适的行后再 `generate`
